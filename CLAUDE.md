# CLAUDE.md - Shopii Project Guidelines

## Code Quality Standards (CRITICAL)
- Be **extremely thorough and comprehensive** with every task. No shortcuts, no half-measures.
- Every change must be complete, correct, and production-ready.
- Read and fully understand all relevant code before making any changes.
- Verify changes don't break existing functionality.
- If a task description says to do N things, do all N things — do not skip items or leave TODOs.

## Git Rules (CRITICAL)
- Commits are allowed locally — commit as needed throughout each task.
- **NEVER** push or merge without explicit user permission.
- When it's time to push/merge, provide the commands for the user to run themselves.
- Branch naming convention: `tyler/SHO-###-short-description`
- Work through tasks one branch at a time: create branch, make changes, commit, then move to next.
